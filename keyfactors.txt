ShotMask / 枪声标注与探测 — 要点速查
（说明：第一层数字 = 大类；第二层 x.y = 该大类下的条目。
  未特别注明时，运行时校准项在 calibrated_detector_params.json。）

────────────────────────────────────────

1 竖线对齐与精炼（refinement）

────────────────────────────────────────

1.1  shot_marker_alignment

     含义：Stage2 出结果后，竖线时间上再对齐到哪种参考。

     取值：

     • onset — 感知起始（包络第一次越过一定比例峰值），竖线通常比主峰早。

     • envelope_peak — 在 marker_refine_window_sec 窗内对齐局部包络最大值（峰顶），最易与波形对眼。

     • none — 不挪动，保留模型/打分给出的原始时刻。



1.2  marker_refine_window_sec

     含义：1.1 所用窗宽（秒）：以当前估计时刻为中心 ± 半窗。窗略大更易兜住圆钝峰顶，过小可能卡在上升沿。



────────────────────────────────────────

2 置信度与「线数量」控制

────────────────────────────────────────

2.1  min_confidence_threshold

     当前值（calibrated_detector_params.json）：0.65

     含义：Stage2 最终列表的置信度硬门槛，低于则丢弃。调高 → 线更少更干净、易漏枪；调低 → 线多。

     （JSON 里人为设定，不是每条视频自动算出最优。）



2.2  annotate_shots：--annotate-loose

     含义：关闭 min_confidence_threshold 过滤（旧行为），候选线会暴增；仅当「宁可多标再删」时用。



2.3  annotate_shots：--shot-nms SEC（默认 0.12）

     含义：SEC 秒内多条检波合并为一条并保留最高置信度；压回声/重复峰。SEC=0 关闭。



────────────────────────────────────────

3 标注数据来源与会话（annotate 行为）

────────────────────────────────────────

3.1  同目录 *cali.txt 是否存在

     含义：若已有 {视频基名}cali.txt 且未被 force_detect 绕过，直接读文件行、不跑机检；仅 # 头无数字 → 0 条线。

     annotate 只认视频所在文件夹的 cali/beep，不从别处搜。



3.2  http://127.0.0.1:8765/annotate.html 与进程

     含义：浏览器连的是「当前正在跑的」Python 会话；未 Ctrl+C 重开则仍可能是上一段视频的状态。

3.3  SHOTMASK_TRAINING_DATA_ROOT（训练 / dataset_split.json 的根目录）

     含义：默认为项目下的 training data/；若数据在别的盘符，可先 set 环境变量再跑 train_*、evaluate_multivideo --use-split、refresh_dataset_split_from_cali、count_data 等。

     例（JSON 里路径为 outdoor/xxx.mp4）：set SHOTMASK_TRAINING_DATA_ROOT=D:\shotmask_data\training data
     （该目录内应有 dataset_split.json 与 outdoor\ 等子文件夹；不要把变量指到单独的 outdoor 子目录，除非你改了 JSON 相对路径写法。）

     实现见 training_data_root.py。



────────────────────────────────────────

4 检测后端与 CNN 输入（模型侧）

────────────────────────────────────────

4.1  use_cnn_only、cnn_gunshot_path、ast 等（校准 JSON）

     含义：为 true 时 Stage2 以 CNN（或 AST）概率作主分数；阈值、竖线对齐仍受第 1–2 类影响。



4.2  CNN Mel 非对称时间窗（detectors/shot_cnn.py 常量，非 JSON）

     名称：SEGMENT_PRE_SEC、SEGMENT_POST_SEC（SEGMENT_DURATION = 两者之和）。

     当前默认：pre = 0.05 s（锚点 t 前），post = 0.08 s（t 后），总长 0.13 s。

     含义：以候选 t 截取 [t−pre, t+post] → Mel → 2D CNN；前短后长，覆盖起跳 + 尾迹/回声。

     改参需与训练一致，否则宜重训。



────────────────────────────────────────

5 Stage1 候选与 Stage2 聚类（声学粗筛层）

────────────────────────────────────────

5.1  cluster_window_sec、candidate_min_dist_ms、mad_k

     含义：控制 onset 候选疏密与簇合并尺度；过小 → 噪声/重复峰多；过大 → 易并掉相邻真枪。

     与 2.x（置信度/NMS）不同层，但同属「量减少 / 误检」相关旋钮。



────────────────────────────────────────

6 评测与参数归档（指标对齐、记在哪儿）

────────────────────────────────────────

6.1  evaluate_multivideo.py：ref 与检出时刻匹配容差 tol

     默认：TOL = 0.06 s（代码常量 evaluate_multivideo.TOL）→ 每条真值时刻与「最近一条检出」的时间差 ≤ 0.06 s 计为 TP（对称距离，非半窗倍乘）。

     命令行：--tol <秒> 覆盖；与训练损失无关，只影响离线 P/R/F1 报告。

     对比模型或不同 checkpoints 时应有意识固定同一 tol；容差放大通常使 P/R/F1 变好看。

     与默认 tol 同步的说明写在 outputs/training_results_latest.json → evaluation 块（machine-readable）。


6.2  outputs/training_history.jsonl

     含义：train_cnn_gunshot 每次跑完追加一行 JSON（含当次超参、样本数、best loss 等）；历史上若曾手写回填，也会带 evaluate_multivideo_match_tolerance_* 等字段。


6.3  「程序员是不是更喜欢用 JSON 记这些？」

     • 记「给程序读」的字段（路径、数字、版本）——JSON（或 YAML/TOML）很常见：一行/一个文件就能被脚本、CI、notebook 直接解析，且多数语言标准库或一行依赖即可读写。

     • 记「给人读」的设计说明、旋钮含义、操作步骤——仍是纯文本 / Markdown 更合适（如此 keyfactors.txt）；二者常并存：.txt/.md 解释「是什么」，.json 保存「具体数值与路径」。

     • JSONL（每行一个 JSON）适合日志式追加（training_history.jsonl），比单个巨大 JSON 数组更易尾部追加、流式处理。
